  <name>Task 1: Build /events listing page with CityFilter client component</name>
  <files>app/events/page.tsx, app/events/CityFilter.tsx</files>
  <action>
    **app/events/page.tsx** — Server Component (no 'use client'):
    - Import `getEvents` from `@/lib/events`
    - Import `Calendar` from lucide-react (size-4 text-stone-400 in heading)
    - Import `CityFilter` and `Suspense` from react
    - DO NOT import or call `requireSubscription` — events are free tier
    - Fetch: `const store = await getEvents()`
    - Filter past events: `const today = new Date().toISOString().split('T')[0]` then `const upcoming = store?.events.filter(e => e.date >= today) ?? []`
    - Page heading: icon + "Events" + count badge (count = upcoming.length)
    - If `store` is null or `upcoming.length === 0`, render empty state:
      - Calendar icon (size-4, text-stone-400) + "No upcoming events" (text-heading font-serif)
      - "We refresh events every Monday. Check back after the next update." (text-caption text-stone-500)
      - If store exists: show "Last updated [store.generatedAt formatted as 'D MMM YYYY']" (.section-label)
    - Wrap `<CityFilter events={upcoming} />` in `<Suspense fallback={<EventsGrid events={upcoming} />}>` — the fallback renders all events unfiltered while client hydrates (prevents useSearchParams flash)
    - Note: `EventsGrid` is an inline helper component within the same file (or inline JSX in fallback) that renders the card grid without filter state — reuse it in both CityFilter and as Suspense fallback

    **app/events/CityFilter.tsx** — Client Component:
    ```
    'use client';
    import { useRouter, useSearchParams } from 'next/navigation';
    ```
    - Props: `{ events: LegalEvent[] }`
    - Read active city: `const activeCity = searchParams.get('city') as EventCity | null`
    - Derive available cities: `CITY_ORDER.filter(c => events.some(e => e.city === c))` where `CITY_ORDER: EventCity[] = ['London', 'Manchester', 'Edinburgh', 'Bristol', 'Other']`
    - Tab bar: "All" tab + one tab per available city. Style matches TabBar.tsx pattern (see interfaces block). Active tab = city matching URL param; "All" active when no param.
    - Clicking a city tab: `router.push('/events?city=' + city)` or `router.push('/events')` for "All"
    - Filtered events: `activeCity ? events.filter(e => e.city === activeCity) : events`
    - Event card grid: `grid grid-cols-1 sm:grid-cols-2 lg:grid-cols-3 gap-4`

    **Event card design (per CONTEXT.md locked decisions):**
    - Compact card: event name, date, city, organiser — NO external links on card
    - Card is a Next.js `<Link href="/events/{event.id}">` — entire card is clickable
    - Coloured event type tag chip: `rounded-chrome px-2 py-0.5 text-label font-medium` + EVENT_TYPE_COLOURS[event.eventType]
    - Eligibility tag: `.section-label` text, text-stone-500
    - Date: formatted as "Tue, 15 Apr" (short weekday + day + month)
    - If `event.time` present: append time e.g. "18:30"
    - Card: `rounded-card border border-stone-200 dark:border-stone-800 hover:border-stone-400 dark:hover:border-stone-600 transition-colors p-4`
    - Card heading: `text-body font-semibold text-stone-900 dark:text-stone-100`
    - City + organiser: `text-caption text-stone-500`
