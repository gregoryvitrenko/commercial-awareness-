    Create app/api/tracker/route.ts. Use a single POST route with an `action` field (add / update / delete) to avoid creating multiple route files. Pattern matches app/api/quiz/route.ts.

    ```typescript

    export async function GET(request: Request): Promise<Response>

    // POST /api/tracker — body: { action: 'add' | 'update' | 'delete', ... }
    export async function POST(request: Request): Promise<Response>
    ```

    Implementation details:

    GET:
    1. Call requireSubscription() — it redirects internally on failure, but in an API route we need to get the userId. Use `auth()` from `@clerk/nextjs/server` after calling `getSubscriptionStatus()` to check access.
       Actually: follow lib/paywall.ts pattern but adapted for API — call `auth()` to get userId, return 401 if no userId, call `isSubscribed(userId)` from lib/subscription.ts, return 403 if not subscribed (unless PREVIEW_MODE=true or ADMIN_USER_ID matches).
    2. Call `getEntries(userId)` from lib/tracker.ts.
    3. Sort by `deadline` ascending (string sort on YYYY-MM-DD is correct).
    4. Return `Response.json({ entries })` with status 200.

    POST body shapes:
    - `{ action: 'add', firm: string, status: TrackerStatus, deadline: string, notes: string }`
      → Generate id with `crypto.randomUUID()`, createdAt/updatedAt as `new Date().toISOString()`, call `addEntry(userId, entry)`.
    - `{ action: 'update', id: string, status?: TrackerStatus, notes?: string }`
      → Call `updateEntry(userId, id, { status, notes })` (only pass defined fields).
    - `{ action: 'delete', id: string }`
      → Call `deleteEntry(userId, id)`.

    All POST variants return `Response.json({ entries })` with the updated array.

    Error handling: return `Response.json({ error: 'Unauthorized' }, { status: 401 })` for no userId, `Response.json({ error: 'Forbidden' }, { status: 403 })` for not subscribed, `Response.json({ error: 'Bad request' }, { status: 400 })` for invalid action or missing fields.

    Auth check helper (inline in route.ts, not imported from paywall.ts since requireSubscription redirects rather than returns):
    ```typescript
    import { auth } from '@clerk/nextjs/server';
    import { isSubscribed } from '@/lib/subscription';

    async function checkAuth(): Promise<{ userId: string } | Response> {
      if (process.env.PREVIEW_MODE === 'true') return { userId: 'preview-dev' };
      const { userId } = await auth();
      if (!userId) return Response.json({ error: 'Unauthorized' }, { status: 401 });
      if (userId === process.env.ADMIN_USER_ID) return { userId };
      const subscribed = await isSubscribed(userId);
      if (!subscribed) return Response.json({ error: 'Forbidden' }, { status: 403 });
      return { userId };
    }
    ```

    Validate `deadline` is a valid YYYY-MM-DD string before saving on add. Validate `firm` max 100 chars, `notes` max 500 chars.
